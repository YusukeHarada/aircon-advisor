# aircon-advisor

その日の外気温・湿度に応じた推奨エアコン設定温度と運転モードを提示するアプリ。

## 概要

最適化の優先順位を次のとおり定義する。

1. 快適性（主目的）：室内の体感を快適域に置く設定温度を算出する
2. 健康（制約）：室内外の温度差の抑制、高湿度時の熱中症予防で算出値を補正する
3. 省エネ（補足）：主推奨を上書きせず別案として併記する

MVPはエアコンを制御せず、推奨設定温度の表示のみを行う（フィードフォワード）。

## 主な機能

- 地点の外気温・湿度の取得とキャッシュ
- 推奨設定温度・運転モード（冷房／暖房／除湿／送風／不要）の算出
- 健康制約による補正、省エネ別案の併記
- 寒がり／暑がりの個人オフセット、「今暑い／寒い」フィードバック

## 技術構成

- Next.js
- Firebase（Firestore）
- 外部気象API：気象庁（無料・APIキー不要）

## セットアップ

```bash
npm install
npm run dev
```

`.env.local` を用意しなくても起動できる（**デモモード**）。Firebase未設定を検知すると、認証・設定・フィードバックはブラウザのlocalStorageに、気象データは気象庁への直接取得（失敗時は固定サンプル値）にフォールバックし、画面を一通り確認できる。データはそのブラウザにのみ保存され、他の端末とは共有されない。

本番運用する場合は、Firebaseプロジェクトを用意して `.env.local` を設定する：

```bash
cp .env.local.example .env.local  # Firebaseプロジェクトの値を設定
```

- `npm test`：推奨ロジック（lib/recommendation）・気象キャッシュ（lib/weather）の単体テストを実行
- `npm run build`：本番ビルド
- `firestore.rules`：Firestoreセキュリティルール（本人限定＋weather_cacheはサーバー書き込みのみ）
